## 2024-12-16 - Prevent PHP Object Injection in Pluggable Queues via Strict Deserialization
**Vulnerability:** Pluggable queue systems (such as `database`, `redis`, `beanstalkd`, `sync`) deserialize jobs stored as payload strings. Because PHP's `unserialize()` was called without any restriction on `allowed_classes`, any malicious payload injected into the queues could trigger PHP Object Injection (POP gadget chain), allowing arbitrary code execution or arbitrary file deletion on the worker server.
**Learning:** In a decentralized queue or RPC architecture where jobs or messages are serialized and passed around, trusting the content of raw payloads for unrestricted `unserialize()` is extremely risky. Since every job payload specifies its expected `job` class name, we can strictly restrict `unserialize()`'s `allowed_classes` option to exactly the expected class name.
**Prevention:** Never use unrestricted `unserialize($data)`. Always pass a strict whitelist of classes using `['allowed_classes' => [$jobClass]]` or `['allowed_classes' => false]` to prevent unexpected object instantiation and magic methods from triggering.
